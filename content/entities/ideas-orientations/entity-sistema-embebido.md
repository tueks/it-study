# Sistema embebido

## Contexto y frontera

Un sistema embebido combina hardware y software bajo restricciones de recursos, tiempo, energía o entorno para controlar o asistir una función definida. Su uso concreto depende del entorno y no elimina la necesidad de validar supuestos.
